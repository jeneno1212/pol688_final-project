# Nonprofit Organizational Use of DEI Language

A longitudinal study of how nonprofit organizations across four NTEE sectors used Diversity, Equity, and Inclusion (DEI) language in their IRS Form 990 filings from 2016 to 2022.

**Author:** Jonathan Espinosa
**Course:** POL 688, University of Arizona (Spring 2026)
**Instructor:** Dr. Yotam Shmargad

---

## Project Overview

This project examines whether and how DEI language in IRS Form 990 filings changed between 2016 and 2022 across 12 nonprofits in four NTEE sectors:

- **Arts & Culture (A):** Met Museum of Art, Lincoln Center, Kennedy Center
- **Education (B):** NAACP LDF, UNCF, Teach For America
- **Civil Rights (R):** ACLU Foundation, National Urban League, HRC Foundation
- **Human Services (P):** United Way, Goodwill Intl, American Red Cross

For each organization, four target years (2016, 2018, 2020, 2022) were sampled, for a target of 48 filings. Four PDFs were unavailable on ProPublica, yielding a final corpus of **44 filings**.

The analysis uses dictionary-based text matching against a 124-word DEI lexicon assembled from the Cornell University DEI Glossary and the NSF leaked trigger word list (2025). The output is a longitudinal panel of DEI word counts by organization, sector, and year.

## How to Reproduce

### Requirements

- **R** (≥ 4.2)
- **RStudio** (recommended)
- A working installation of **Tesseract OCR** (used by the `tesseract` R package for OCR of scanned PDFs). On Windows, the package will bundle this; on macOS, install via `brew install tesseract`.
- A working internet connection (Script 2 calls the ProPublica API)
- Approximately 5 GB of free disk space for the full PDF + text corpus

### R Packages

Install all required packages with:

```r
install.packages(c(
  "tidyverse",   # data wrangling + ggplot2
  "httr",        # API requests
  "jsonlite",    # JSON parsing
  "pdftools",    # text extraction from PDFs
  "tesseract",   # OCR for scanned/image-based PDFs
  "stringr",     # string operations + term counting
  "png"          # image handling for OCR pipeline
))
```

### Reproduction Steps

The scripts are numbered and meant to be run in order. Each script writes its outputs to `Data/` or `Results/` so the next script can pick them up.

> **Working directory:** Each script currently uses an absolute path (`setwd("C:/Users/jenen/JE/2026/Arizona/Spring 2026/POL 688/Final Project/Data")`). Before running, **update each `setwd()` call to point to your local clone of this repository.**

#### Step 1 — Build the sample (`01_build_sample.R`)

Downloads the ProPublica filing index for all 12 EINs across all years, then filters down to orgs with filings present in at least 7 of the target years to ensure a balanced longitudinal sample.

**Inputs:** none (hits ProPublica API)
**Outputs:** `Data/longitudinal_sample.csv`, `Data/index_all_years.csv`

#### Step 2 — Build the PDF URL log (`02_build_url_log.R`)

Queries the ProPublica Nonprofit Explorer API once per EIN and extracts the PDF URL for each of the four target years (2016, 2018, 2020, 2022). Writes a CSV log of URLs.

**Inputs:** none (hits ProPublica API)
**Outputs:** `Data/pdf_url_log.csv`

**Manual step:** Open `Data/pdf_url_log.csv`. Each row has a `pdf_url` and a `save_as` filename. From a logged-in browser session, click each URL and save each PDF to `Data/PDFs/` using the `save_as` filename (`{EIN}_{YEAR}.pdf`). See the *Challenges* section below for why this step is manual rather than scripted.

#### Step 3 — Extract text via OCR (`03_extract_text.R`)

Loops through every PDF in `Data/PDFs/`, uses `pdftools::pdf_ocr_text()` (which calls Tesseract under the hood) to extract text page-by-page, and writes one `.txt` file per filing to `Data/Text/`. Also builds a text corpus index recording word counts.

**Inputs:** `Data/PDFs/*.pdf`
**Outputs:** `Data/Text/{EIN}_{YEAR}.txt`, `Data/text_corpus_index.csv`

> **Heads-up:** OCR is slow. Expect roughly 5–10 minutes per 990 (they run 50–200+ pages). Plan for several hours of total runtime on the full corpus.

#### Step 4 — Score DEI language (`04_score_dei.R`)

Loads each text file, lowercases it, and counts occurrences of each term in the 124-word DEI lexicon using `stringr::str_count()`. Produces two outputs: an org-year roll-up and a term-level long-format file.

**Inputs:** `Data/Text/*.txt`
**Outputs:** `Data/dei_scores.csv`, `Data/dei_term_counts.csv`

#### Step 5 — Visualize (`05_visualize.R`)

Generates the three figures used in the presentation:

- **Plot 1** — org-level trend lines, faceted by NTEE sector
- **Plot 2** — grouped bar chart of sector averages by year
- **Plot 3** — HRC Foundation vs. Civil Rights sector average

**Inputs:** `Data/dei_scores.csv`
**Outputs:** `Results/plot1_org_trends_by_sector.png`, `Results/plot2_sector_averages_by_year.png`, `Results/plot3_hrc_vs_civil_rights.png`

## Data Sources

- **IRS Form 990 PDFs** — [ProPublica Nonprofit Explorer](https://projects.propublica.org/nonprofits/) (free, public). Accessed via the public API at `https://projects.propublica.org/nonprofits/api/v2/organizations/{EIN}.json`.
- **DEI lexicon (124 terms)** — assembled from:
  - The Cornell University DEI Glossary
  - The leaked NSF DEI trigger word list (reported in 2025 press coverage of federal grant review changes)
- **Sector classification** — IRS NTEE codes assigned by ProPublica.

---

## Key Findings

- DEI word counts ranged from single digits (Kennedy Center, 2016) to nearly 200 (HRC Foundation, 2022).
- The 2020 filing year shows the largest average increase in DEI language across all four sectors.
- Sector matters: Civil Rights organizations use roughly 3–4 times more DEI language than Arts & Culture organizations.
- HRC Foundation shows the steepest upward trend in the corpus.

## Challenges

The single biggest challenge was getting the PDFs off ProPublica programmatically. Several approaches were attempted and abandoned:

1. **RSelenium** — dropped due to a missing Java dependency that could not be resolved on the project machine.
2. **`chromote` scripts (versions 2–5)** — multiple iterations failed for different reasons: incorrect session initialization, download-event detection issues, Chrome windows not opening visibly, and persistent timeouts. Fixes that were tried but did not resolve the problem included non-headless mode, CDP network interception, and synchronous navigation.
3. **`httr::GET()` direct download** — returned HTTP 403 Forbidden. ProPublica requires real browser session cookies and rejects unauthenticated programmatic requests.
4. **Manual cookie extraction** — proposed as a fallback (using a browser extension like "Cookie-Editor" to inject session cookies into `httr` requests) but ultimately not necessary because of the simpler approach below.

**Resolution:** Rather than fight ProPublica's bot protections, I split the task. Script 2 uses the ProPublica *API* (which is permissive) to harvest the PDF URLs and write them to a log. Then I opened each URL manually from a logged-in browser session and saved the PDFs. This was about an hour of clicking, but it was reliable and reproducible — and the URL log makes the manual step systematic rather than ad hoc.

A secondary challenge worth noting: many of ProPublica's 990 PDFs are scanned images, not searchable text. This made OCR (via `pdftools` + `tesseract`) a necessary step rather than an optional one, and added significant runtime to Script 3.
